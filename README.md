## 什么是搬瓦工 KVM 架构，和老的 OpenVZ 有什么区别

KVM（Kernel-based Virtual Machine）是一种完整的虚拟化方案，每台 VPS 拥有独立的虚拟内核，可以自由替换操作系统、加载自定义内核模块。早期搬瓦工主推 OpenVZ 架构，内核共享，不支持 BBR 这类需要内核级别支持的加速算法。搬瓦工现在全面切到 KVM，就是 KiwiVM 面板里能直接一键重装系统、一键内置 BBR 的原因。

**BBR 是什么**：Google 推出的 TCP 拥塞控制算法，和传统算法相比，不再把丢包作为拥塞信号，而是通过估算最大带宽和最小延迟来调整发送窗口。用下来的直观感受是网速明显稳了，丢包少了，尤其在高峰期线路拥堵时差距更明显。

## 搬瓦工 KVM 一键安装系统：完整步骤

确认 VPS 是 KVM 架构（搬瓦工现售套餐全是 KVM，无需特别确认），然后按下面步骤操作：

1. **登录 KiwiVM 控制面板**：进入搬瓦工后台，Services → My Services，找到你的 VPS，点击 KiwiVM Control Panel
2. **停止 VPS 运行**：左侧菜单点 Stop，等状态变为 Stopped
3. **进入系统安装**：左侧找到 Install new OS，点击进入
4. **选择系统版本**：从下拉列表选择操作系统——带 `bbr` 后缀的版本（如 `centos-7-x86_64-bbr`）内置 BBR 加速，选这类直接省掉后续手动开 BBR 的步骤；Ubuntu 24.04 LTS 内核本身已经包含 BBR 模块，装完验证一下即可
5. **勾选同意协议，点击 Reload**：确认操作，等待重装完成（通常 2-5 分钟）
6. **记录新的 SSH 端口和 root 密码**：重装后端口和密码会变，面板会显示，只显示一次，记好
7. **验证 BBR 是否生效**：SSH 连接后输入 `lsmod | grep bbr`，输出中出现 `tcp_bbr` 说明已开启

注意：重装会清空 VPS 硬盘所有数据。有重要文件先在 KiwiVM 里做快照备份再操作。

## BBR 验证与手动开启（系统不带 bbr 后缀时）

装完系统 SSH 连进去，先检查 BBR 是否已经运行：

bash
lsmod | grep bbr


如果输出包含 `tcp_bbr`，直接跳过，BBR 已经在跑了。

如果什么都没有，说明需要手动开启。搬瓦工 KVM 方案可以执行：

bash
echo "net.core.default_qdisc=fq" >> /etc/sysctl.conf
echo "net.ipv4.tcp_congestion_control=bbr" >> /etc/sysctl.conf
sysctl -p


执行后再跑一遍 `lsmod | grep bbr` 验证。如果内核版本过低（低于 4.9），需要升级内核后再操作——这也是为什么建议直接选 Ubuntu 24.04 或带 bbr 标记的系统，省掉这一步。

用了两周下来感受是：开了 BBR 之后连接稳了很多，以前晚高峰偶尔会有卡顿，现在基本感觉不到。
